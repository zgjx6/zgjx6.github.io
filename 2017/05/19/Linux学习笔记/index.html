3.命令行BASH的基本操作

  • shell分为CLI和GUI
  • $-普通用户;#-ROOT用户
  • 用户名@主机名 当前目录;~表示当前用户主目录
  • 命令组成:命令 选项 参数
  • hostname:主机名
  • uname:当前系统信息,底层内核信息
    • -r:内核版本号
    • -a:全部
    • --all与-a作用相同
  • 应用名直接打开应用:firefox
    • firefox &:后台运行
  • ctrl+c:终止当前命令
  • Tab自动补全命令或文件名
    • 两次Tab可查找所有
  • clear:清除当前屏幕
  • 历史命令:
    • 上下键可查看以前的命令
    • history:所有历史命令
      • -c:删除历史命令
    • !!:重复前一个命令
    • !字符:重复前一个以”字符”开头的命令
    • !num:按照历史记录的序号执行命令
    • !?abc:重复之前包含abc的命令
    • !-n:重复N个命令之前的那个命令
    • ctrl+r:搜索历史命令
    • esc+.:调用上一个命令的参数
  • Bash支持通配符
    • *:零个或多个
    • ?:任意一个
    • [0-9]:一个数字范围
    • [abc]:列表里任何字符
    • [^abc]:列表以外字符
  • 用户
    • su - 用户名:切换用户
    • su -:Root用户.加-切换终端;不加仅切换用户
    • sudo:使用管理员身份运行命令
    • exit:退出ROOT用户
    • id:显示当前用户信息
    • passwd:修改密码
  • 作业管理
    • 命令后加&符号:后台运行
    • nohup:不挂断地运行命令
    • ctrl+z:暂停后台任务
    • jobs:查看后台作业
    • sleep n:停n秒
    • bg n:恢复运行
    • fg n:前台运行
    • pgrep 进程名:-l 同时显示进程名和PID
      • pgrep相当于 ps -eo pid,cmd | awk ‘{print $1,$2}’ | grep KeyWord
    • ps:显示当前进程的状态
      • a 显示所有进程
      • -a 显示同一终端下的所有程序
      • -A 显示所有进程
      • c 显示进程的真实名称
      • -N 反向选择
      • -e 等于“-A”
      • e 显示环境变量
      • f 显示程序间的关系
      • -H 显示树状结构
      • r 显示当前终端的进程
      • T 显示当前终端的所有程序
      • u 指定用户的所有进程
      • -au 显示较详细的资讯
      • -aux 显示所有包含其他使用者的行程
      • -C<命令> 列出指定命令的状况
      • --lines<行数> 每页显示的行数
      • --width<字符数> 每页显示的字符数
      • --help 显示帮助信息
      • --version 显示版本显示
    • top:动态的显示进程信息
  • linux上进程有5种状态:
    1. 运行(正在运行或在运行队列中等待)
    2. 中断(休眠中, 受阻, 在等待某个条件的形成或接受到信号)
    3. 不可中断(收到信号不唤醒和不可运行, 进程必须等待直到有中断发生)
    4. 僵死(进程已终止, 但进程描述符存在, 直到父进程调用wait4()系统调用后释放)
    5. 停止(进程收到SIGSTOP, SIGSTP, SIGTIN, SIGTOU信号后停止运行运行)

5.Linux文件基本操作管理

  • cp 源文件(夹) 目标文件(夹):复制文件、目录
    • -r:递归复制整个目录(复制文件夹需要)
    • -v:显示详细信息
    • -b:为每个已存在的目标文件创建备份
    • -a; 该选项通常在拷贝目录时使用。它保留链接、文件属性，并递归地拷贝目录，其作用等于dpR选项的组合。
    • -i:询问
    • -f:强制
    • -n:不要覆盖已存在的文件
  • mv 文件 目标目录:移动文件;指定文件名可用来重命名
    • -b：若需覆盖文件，则覆盖前先行备份。
    • -f：force强制的意思,如果目标文件已经存在,不会询问而直接覆盖
    • -i：若目标文件 (destination) 已经存在时，就会询问是否覆盖！
  • rm:删除文件或目录
    • -r:递归删除整个目录(删除文件夹需要)
    • -i:交互式
    • -f:强制删除,没有警告提示
  • mkdir:创建目录
    • -m:设定权限.例:mkdir -m 777 test3
    • -p:可以是一个路径名称.例:mkdir -p test2/test22
  • rmdir:删除空目录,不能删除非空文件夹

  • 需要超级权限；
  • fdisk -l:列出所有磁盘及分区信息
  • fdisk /dev/sda:可对目标磁盘进行分区

  • /proc/partitions:也可查看分区信息
  • m:帮助信息
  • n:创建新分区

  • w:保存修改并退出

11.Linux文件系统

  • 没有文件系统的设备称为裸（RAW）设备
  • 创建文件系统的过程称为格式化
  • 常用：WINDOWS:ntfs;LINUX:Ext3，Ext4
  • mke2fs -t ext4 /dev/sda3:创建Ext4文件系统
    • -b 4096（2048）:指定文件系统块大小，每次读写最小单位，字节
    • -c:建立文件系统时检查坏损块
    • -L King:指定卷标
    • -j:建立文件系统日志
  • mkfs.ext4 /dev/sda3:也可创建，参数少，不能精细化
  • dumpe2fs /dev/sda3:查看文件系统详细信息
  • e2label /dev/sda3 标签:为文件系统添加标签
  • fsck /dev/sda3:检查并修复损坏的文件系统，必须卸载
    • -y:不提示直接修复
    • -t:指定文件系统类型
    • 损坏数据会被放入lost+found目录
    • 系统启动时会进行fsck操作

12.Linux文件系统挂载管理

  • mount /dev/sda3 /mnt:将格式化好的磁盘或分区挂载到一个目录上
    • 不带参数将会显示所有以挂载的文件系统
    • -t:指定文件系统的类型
    • -o:指定挂载选项，多选项逗号分隔
      • ro,rw:以只读或读写形式挂载，默认rw
      • sync:不使用缓存，直接写入磁盘
      • async:使用缓存，默认
      • noatime:每次访问文件时不更新访问时间
      • atime:更新访问时间
      • remount:重新挂载文件系统
  • umount 文件系统/挂载点:卸载文件系统，相当于弹出
    • 如果出现device is busy,正在使用无法卸载
    • fuser -m /mnt:查看使用文件系统的进程
    • lsof /mnt:查看正在被使用的文件系统
  • /etc/fstab配置文件:定义需要自动挂载的文件系统，每一行代表一个挂载配置
    • 需要挂载的设备、挂载点、文件系统、挂载选项、dump/fsck相关选项
    • 要挂载的设备也可使用LABEL进行识别，LABEL=卷标取代/dev/sda3
    • mount -a:挂载fstab中自动挂载项
  • 对文件系统进行修改必须先卸载

15.Linux权限机制

  • r 读权限（文件夹必须拥有x权限才行）
  • w 写权限
  • x 执行
    • 没有权限
  • UGO模型:
    • 第一位代表文件类型
      • d 文件夹
      • - 文件
      • l 链接
    • user
    • group
    • other
  • chown 用户名 文件（夹）名：修改文件所属用户
    • -R：递归修改目录下的所有文件的所属用户
  • chgrp 用户名 文件（夹）名：修改文件所属组
    • -R：递归修改目录下的所有文件的所属组
  • chmod 模式 文件：修改文件权限
    • u,g,o分别代表用户、组和其他，a代表全部
    • +、-代表添加或删除相应权限
    • r、w、x代表三种权限。4、2、1可分别代替，权限应为数字之和,使用数字时同时修改UGO
    • 示例：chmod ug+rw aabbcc.txt、chmod 660 aabbcc.txt

16.Linux扩展权限

  • 默认权限：
    • 每个终端都有一个umask属性，用来确定新建文件和目录的默认权限
    • umask使用数字权限方式表示，如022
    • 目录默认：777-umask.文件默认：666-umask
    • umask:查看umask值。默认普通用户002，root用户022
      • 加上一个值，可以修改。如umask 022:将当前用户默认值改为022
  • 特殊权限：
    • suid 以文件所属用户身份执行，而非执行文件的用户。对目录无影响
    • chmod u+s aabbcc.txt:设置suid。数字为4
    • sgid 文件：以文件所属组身份执行。目录 该目录新建文件所属组与目录所属组相同
    • chmod g+s aabbcc.txt:设置sgid。数字为2
    • sticky 对文件无影响。对目录有写入权限的人仅可删除其用有的文件，无法删除其他用户拥有的文件
    • chmod o+s aabbcc.txt:设置sticky。数字为1
    • 例：chmod 4755 aabbcc.txt

20.Linux命令行文本处理工具

  • 基于关键字搜索文本:grep
    • -i:忽略大小写

    • -d:指定分隔字符（默认TAB）（-d:以冒号为分隔符）
    • -f:指定输出的列号（-f1只显示第三列）
    • -c:基于字符进行切割（-c2-6显示第二到第六个字符）
  • 统计文本基本信息:wc(wordcount)
    • -l:行数
    • -w:单词
    • -c:字节
    • -m:字符
  • 文本排序:sort
    • -r:倒序

    • init改变当前运行级别

SecureCRT

  • ps:显示当前在系统运行的进程
    • -e:显示每个现在运行的进程
    • -f:生成一个完全的列表
  • rz:选择要上传的文件，确定
  • sz:指定文件名,enter敲，即下载到了secureCRT/download目录下
  • df:查看磁盘使用情况
    • a 全部文件系统列表
    • h 方便阅读方式显示
    • H 等于“-h”，但是计算式，1K=1000，而不是1K=1024
    • i 显示inode信息
    • k 区块为1024字节
    • l 只显示本地文件系统
    • m 区块为1048576字节
    • --no-sync 忽略 sync 命令
    • P 输出格式为POSIX
    • --sync 在取得磁盘信息前，先执行sync命令
    • T 文件系统类型
  • du:查看目录大小
    • a或-all 显示目录中个别文件的大小。
    • b或-bytes 显示目录或文件大小时，以byte为单位。
    • c或--total 除了显示个别目录或文件的大小外，同时也显示所有目录或文件的总和。
    • k或--kilobytes 以KB(1024bytes)为单位输出。
    • m或--megabytes 以MB为单位输出。
    • s或--summarize 仅显示总计，只列出最后加总的值。
    • h或--human-readable 以K，M，G为单位，提高信息的可读性。
    • x或--one-file-xystem以一开始处理时的文件系统为准，若遇上其它不同的文件系统目录则略过。
    • L<符号链接>或--dereference<符号链接> 显示选项中所指定符号链接的源文件大小。
    • S或--separate-dirs 显示个别目录的大小时，并不含其子目录的大小。
    • X<文件>或--exclude-from=<文件> 在<文件>指定目录或文件。
    • --exclude=<目录或文件> 略过指定的目录或文件。
    • D或--dereference-args 显示指定符号链接的源文件大小。
    • H或--si 与-h参数相同，但是K，M，G是以1000为换算单位。
    • l或--count-links 重复计算硬件链接的文件。
    • --max-depth=1:只显示当前目录
  • pstree:以树的形式显示的server进程
    • -p:显示每个进程的 PID
    • -u:显示启动该进程的用户名
  • ifconfig:查看IP
