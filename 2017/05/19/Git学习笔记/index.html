Git学习笔记

发表于 2017-05-19 | 分类于 学习
这是学习git的笔记

设置

设置name:git config --global user.name "Your Name"
设置邮箱:git config --global user.email "email@example.com"

创建版本库

1.cd到某目录
2.git init:创建仓库
3.git add readme.txt file2.txt:添加文件或者目录;-f强制添加
4.git commit -m "wrote a readme file":提交并添加注释

版本控制

git status:查看仓库当前状态
git diff readme.txt:查看文件变更
git diff HEAD -- readme.txt:查看工作区和版本库里面最新版本的区别
git log:查看提交日志;--pretty=oneline:简化
HEAD:当前版本;HEAD^:上一个版本;HEAD~100:上100个版本
git reset --hard HEAD^:回退到上个版本
git reset --hard 3628164:回到指定版本
git reflog:查看命令记录
git checkout -- file:丢弃工作区的修改
git reset HEAD file:把暂存区的修改撤销掉（unstage），重新放回工作区
git rm -r --cache test.txt:删除文件或目录

远程仓库

ssh-keygen -t rsa -C “youremail@example.com”:一路回车，使用默认值.用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。然后在Github设置中SSH Keys添加id_rsa.pub文件的内容.
git remote add origin git@github.com:accountname/learngit.git:添加远程仓库
git push -u origin master:推送本地master到远程.
git pull:从远程下载最新版本并合并
       -u:由于远程库是空的，我们第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。
当你第一次使用Git的clone或者push命令连接GitHub时，会得到一个警告.
git clone git@github.com:accountname/gitskills.git:从远程克隆

分支管理

git checkout -b dev:创建并切换到dev分支
git branch dev:创建分支
git checkout dev:切换分支
git branch:查看分支,当前分支前面会标一个*号
git merge dev:合并dev分支到当前分支,Fast-forward“快进模式”
git branch -d dev:删除dev分支
git merge --no-ff -m “merge with no-ff” dev:强制禁用Fast forward模式,合并后历史有分支,可查看
git stash:保留现场
git stash list:查看工作现场
git stash apply:恢复现场但不删除现场
git stash drop:删除现场
git stash pop:恢复并删除现场
git stash apply stash@{0}:恢复指定现场
git branch -D feature-vulcan:强行删除分支
git remote:查看远程库的信息
git remote -v:显示更详细的信息
git checkout -b dev origin/dev:创建远程origin的dev分支到本地
git branch --set-upstream dev origin/dev:指定本地dev分支与远程origin/dev分支的链接

标签管理

git tag :打一个新标签
git tag:查看所有标签
git tag v0.9 6224937:给某次提交打标签
标签不是按时间顺序列出，而是按字母排序的
git show :查看标签信息
      -u:-a指定标签名;-m指定说明文字;-s用私钥签名一个标签
(签名采用PGP签名，因此，必须首先安装gpg（GnuPG），如果没有找到gpg，或者没有gpg密钥对，就会报错);-d删除标签
git push origin :推送某个标签到远程
git push origin --tags:一次性推送全部尚未推送到远程的本地标签
git push origin :refs/tags/v0.9:远程删除标签

自定义Git

git config --global color.ui true:让Git显示颜色
git check-ignore -v App.class:检查.gitignore是否忽略该文件
git config --global alias.st status:为status配置别名st

查看记录:git log --graph --pretty=oneline --abbrev-commit
