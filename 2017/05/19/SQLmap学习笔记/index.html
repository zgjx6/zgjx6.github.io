SQLmap学习笔记

发表于 2017-05-19 | 分类于 学习
这是学习sqlmap的笔记

常用参数

参数 说明 示例及说明
-u 扫描指定URL -u 172.26.4.145/test/get?sql=jzeng
-p 扫描指定注入点(参数) -p sql
--skip 跳过某些参数
-r 扫描指定文件，从文件中获取扫描对象及参数
--cookie 通过-u扫描指定url且需有会话标识时需要此参数
--data 指定URL相应的POST参数
--flush-session 清除当前扫描目标会话文件 重新扫描时使用
--dbms 指定数据库类型 --dbms “Mysql”
--dbs 爆数据库
-D 指定数据库 -D xice
--tables 列出数据库中的表
-T 指定表名 -T user
--columns 列出表中字段
-C 指定列名 -C user,pwd --dump
--dump 将结果导出
--start 指定开始的行 --start 1
--stop 指定结束的行 --stop 10
--current-db web当前使用的数据库
--current-user web数据库使用账户
--users 列出sqlserver所有用户
--passwords 数据库账户与密码
--threads 最大的HTTP（S）请求并发量（默认为1） --threads 5
--proxy 设置代理 --proxy=http://172.17.18.80:8080
--version 显示程序的版本号并退出
-h,--help 显示此帮助消息并退出
-v VERBOSE 详细级别：0-6（默认为1）
-d DIRECT 直接连接到数据库
--level=LEVEL 执行测试的等级（1-5，默认为1） 数值>=2的时候会检查cookie，当>=3的时候将检查User-agent和Referer
--risk=RISK 执行测试的风险（0-3，默认为1） 共有四个风险等级，默认是1会测试大部分的测试语句，2会增加基于事件的测试语句，3会增加OR语句的SQL注入测试。
--tamper 使用给定的脚本（S）篡改注入数据 base64encode.py为使用base64编码参数

示例

sqlmap -u 172.26.4.145/test/get?sql=jzeng

其他说明

注入参数sql为GET注入，注入类型有四种分别为：boolean-based blind、error-based、stacked queries、inline query。

其中有若干询问语句，需要用户输入[Y/N]，如果你懒得输入或者不懂怎么输入可以让程序自动输入，只需添加一个参数即可，命令如下：--batch

文件

参数：-r
可通过使用Fiddler工具进行请求抓包，查看其Raw内容获取。
或使用chrome查看请求，复制request Headers。
